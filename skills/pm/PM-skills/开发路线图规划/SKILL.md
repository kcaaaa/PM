# 开发路线图规划 · Base 阶段进度

> 路线图不是大任务表，而是按 `RQ → VR → PD → TD → FT → IMP → TEST` 链路组织的阶段进度。

---

## 1. 输入

- `Base/00-项目总览/项目目标.md`
- `Base/00-项目总览/项目结构基线.md`
- `Base/01-需求管理/RQ-*`
- `Base/05-功能管理/FT-*`
- 已有 `Base/06-实现管理/IMP-*` 与代码结构

---

## 2. 工作流程

1. 扫描已确认 `RQ-*` 与 `FT-*`。
2. 判断依赖关系：地基能力、核心闭环、增强能力。
3. 生成 2-3 个路线图方案：MVP 优先、完整功能优先、低风险渐进。
4. 推荐当前最优路线图。
5. 人类确认后写入 `Base/00-项目总览/阶段进度.md`。
6. 为每个待开发功能指向对应 `IMP-*` 创建条件。

---

## 3. 输出文件

```
Base/00-项目总览/阶段进度.md
```

详细产物模板见：`./ROADMAP-TEMPLATE.md`

---

## 5. 禁止事项

- 禁止路线图脱离 `RQ / FT`。
- 禁止把所有任务写进无追溯关系的大清单。
- 禁止未确认优先级就进入实现。
- 禁止阶段进度脱离 `Base/00-项目总览/阶段进度.md`。
