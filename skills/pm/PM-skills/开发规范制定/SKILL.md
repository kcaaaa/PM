# 开发规范制定 · Base 执行约束

> 开发规范是后续所有 `IMP-*` 的执行约束，必须与技术基线、项目结构和 Base 对象包追溯规则一致。

---

## 1. 输入

- `Base/00-项目总览/技术基线.md`
- `Base/00-项目总览/项目结构基线.md`
- `Base/00-项目总览/决策记录.md`
- 已确认的核心 `FT-*` 与后续 `IMP-*` 约束

---

## 2. 工作流程

1. 读取技术基线与项目结构基线。
2. 结合代码语言、框架、包管理、测试工具生成规范。
3. 明确 AI 编码前必须读取的 Base 对象包。
4. 明确代码实现必须绑定 `RQ / FT / IMP`。
5. 人类确认后写入 `Base/00-项目总览/开发规范.md`。

---

## 3. 输出文件

```
Base/00-项目总览/开发规范.md
```

详细产物模板见：`./STANDARD-TEMPLATE.md`

---

## 4. 开发规范.md 必须包含

- 语言与运行时要求。
- 代码风格与格式化规则。
- 命名约定。
- 目录依赖方向。
- Git / 提交规范。
- 测试规范。
- 安全编码规则。
- AI 协作规则。
- Base 追溯规则。

---

## 5. AI 协作规则必须写明

```
编码前必须读取：
- 对应 RQ
- 对应 PD
- 对应 TD
- 对应 FT
- 对应 IMP
- 如为变更，读取对应 CR

编码后必须更新：
- IMP/实现完成记录.md
- TEST 测试验收包
```

---

## 6. 禁止事项

- 禁止生成与技术基线不一致的规范。
- 禁止遗漏 Base 追溯规则。
- 禁止用通用最佳实践替代项目约束。
- 禁止开发规范脱离 `Base/00-项目总览/开发规范.md`。
