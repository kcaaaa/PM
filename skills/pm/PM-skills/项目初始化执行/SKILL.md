# 项目初始化执行 · Base 驱动脚手架

> 初始化只把已确认的 Base 基线转化为代码骨架，不做未定义的技术决策。

---

## 1. 前置条件

必须存在：

- `Base/00-项目总览/技术基线.md`
- `Base/00-项目总览/项目结构基线.md`
- `Base/00-项目总览/开发规范.md`
- `Base/00-项目总览/阶段进度.md`

若缺失任一文件，不得初始化，应退回对应 PM 子 Skill 补齐。

---

## 2. 工作流程

1. 检查项目是否已初始化：`.git`、`package.json`、`go.mod`、`pyproject.toml` 等。
2. 若已初始化：不创建或覆盖骨架，只生成初始化检查记录。
3. 若未初始化：读取 `技术基线.md` 和 `项目结构基线.md`。
4. 按已确认目录树创建项目结构。
5. 创建基础文件：`.gitignore`、`README.md`、`.env.example` 等。
6. 初始化版本控制和包管理配置。
7. 生成 `Base/00-项目总览/初始化记录.md`。
8. 更新 `Base/00-项目总览/阶段进度.md`。

---

## 3. 输出

```
Base/00-项目总览/初始化记录.md
```

初始化记录必须包含：

- 使用的技术基线。
- 使用的项目结构基线。
- 创建的目录和文件。
- 跳过项和原因。
- 风险和后续建议。

详细产物模板见：`./INIT-TEMPLATE.md`

---

## 4. 安全约束

- 删除、覆盖、迁移已有文件前必须获得人类明确确认。
- 不得创建 Base 未定义的技术栈或目录结构。
- 不得把初始化记录写入 `Base/` 之外的临时文档作为唯一记录；Base 必须同步。
- 项目骨架就绪后，按 `阶段进度.md` 进入 `FT → IMP → TEST` 链路。
