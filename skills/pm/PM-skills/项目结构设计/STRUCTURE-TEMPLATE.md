# 项目结构基线模板

适用文件：`Base/00-项目总览/项目结构基线.md`

```md
# 项目结构基线

## 1. 上级依据

- 项目目标：`Base/00-项目总览/项目目标.md`
- 技术基线：`Base/00-项目总览/技术基线.md`
- 关联需求：`Base/01-需求管理/RQ-*`
- 关联功能：`Base/05-功能管理/FT-*`

## 2. 推荐目录树

```text
/
├── Base/                  # AI 项目认知数据库
│   ├── 00-项目总览/
│   ├── 01-需求管理/
│   ├── 02-验证管理/
│   ├── 03-产品设计/
│   ├── 04-方案设计/
│   ├── 05-功能管理/
│   ├── 06-实现管理/
│   ├── 07-反馈管理/
│   ├── 08-变更管理/
│   └── 09-测试验收/
├── docs/                  # 面向人类的说明文档
├── src/                   # 源代码
│   ├── modules/           # 基于 FT 功能对象拆分的业务模块
│   └── shared/            # 通用组件、工具、基础能力
├── tests/                 # 测试代码
└── [配置文件]
```

## 3. 关键目录说明

| 目录 | 职责 | 约束 |
|---|---|---|
| `Base/` | 需求、方案、功能、实现、测试与变更追溯 | 不存放真实源码 |
| `src/modules/` | 业务模块 | 原则上映射 `FT-*` |
| `src/shared/` | 共享能力 | 不承载具体业务意图 |
| `tests/` | 测试代码 | 应映射 `TEST-*` |

## 4. 文件放置规则

- 页面 / 路由：放在框架约定目录。
- 业务逻辑：放在 `src/modules/[功能模块]`。
- 通用组件：放在 `src/shared/components`。
- 通用工具：放在 `src/shared/utils`。
- 测试文件：与 `TEST-*` 对象建立对应关系。

## 5. 与 Base 的映射

| Base 对象 | 代码位置 | 说明 |
|---|---|---|
| RQ-* | 无直接代码 | 需求来源 |
| FT-* | `src/modules/[模块]` | 功能边界 |
| IMP-* | 具体修改文件索引 | 实现任务与代码映射 |
| TEST-* | `tests/` 或测试工具输出记录 | 测试验收 |
| CR-* | 增量修改文件索引 | 变更来源 |

## 6. 协作规则

- 用户可以只描述业务诉求，不需要指定源码位置。
- AI 必须根据本结构基线自行定位实现位置。
- 修改完成后，必须把实际修改文件、验证方式、验证结果写回对应 `IMP-*` / `TEST-*`。
```
