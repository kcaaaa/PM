# 功能编码实现 · 基于 IMP 的执行规则

> 编码不是从用户一句话开始，而是从已确认的 `IMP-*` 实现包开始。

---

## 1. 前置条件

编码前必须存在：

- `Base/01-需求管理/RQ-*`
- `Base/03-产品设计/PD-*`
- `Base/04-方案设计/TD-*`
- `Base/05-功能管理/FT-*`
- `Base/06-实现管理/IMP-*`

且 `IMP/实现任务清单.md` 已明确本次实现范围、任务拆解、验收标准。

---

## 2. 执行流程

1. 读取当前 `IMP-*` 实现包。
2. 声明本次代码修改对应的 `RQ / FT / IMP`。
3. 按 `实现任务清单.md` 逐项执行。
4. 涉及数据库、接口、UI 时按需加载 DB / UI / Code Skill。
5. 每完成一项任务，更新实现状态。
6. 完成后更新 `实现完成记录.md`。
7. 创建或更新对应 `TEST-*` 测试验收包。

---


---

## 4. 变更处理

如果编码过程中发现需求、设计或技术方案需要调整：

1. 停止继续扩大实现。
2. 创建或更新 `CR-*` 变更包。
3. 输出影响分析与 2-4 个处理策略。
4. 等待人类确认后再继续。

---

## 5. 禁止事项

- 禁止无 `IMP` 直接编码。
- 禁止无 `RQ / FT` 的代码修改。
- 禁止私自改变产品意图。
- 禁止用代码实现替代需求确认。
- 禁止完成后不更新 Base 实现记录。
