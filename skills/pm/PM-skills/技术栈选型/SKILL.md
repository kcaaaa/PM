# 技术基线选型 · Base 技术约束

> 技术栈不是孤立文档，而是项目级技术基线，服务于后续每个 `TD-*` 与 `IMP-*`。

---

## 1. 输入

- `Base/00-项目总览/项目目标.md`
- `Base/00-项目总览/决策记录.md`
- 首批已确认 `Base/01-需求管理/RQ-*`
- 现有代码结构与团队约束（如有）

---

## 2. 工作流程

1. 读取产品目标、核心需求、非目标范围与约束。
2. 输出 2-4 套技术基线候选方案。
3. 每套方案评估实现复杂度、代码开发量、代码适配度、维护成本、扩展性。
4. 推荐当前最优技术基线。
5. 人类确认后写入 `Base/00-项目总览/技术基线.md`，并更新 `决策记录.md`。

---

## 3. 输出文件

```
Base/00-项目总览/
├── 技术基线.md
└── 决策记录.md
```

详细产物模板见：`./TECH-TEMPLATE.md`

---

```
## 5. 后端技术

```

---

## 5. 禁止事项

- 禁止只因流行选择技术栈。
- 禁止不结合 `Base` 需求链路推荐技术。
- 禁止无候选方案与推荐理由就落定技术基线。
- 禁止技术基线与后续 TD/IMP 脱节。
