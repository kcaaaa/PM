# 产品基线生成 · Base 项目总览

> 产品基线不是单一 `产品概述.md`，而是 `Base/00-项目总览/` 中的项目目标、状态看板、阶段进度与决策记录。

---

## 1. 输入

- 已确认的 `Base/01-需求管理/RQ-*`。
- `RQ/AI候选方案.md` 与 `RQ/人类确认记录.md`。
- 用户确认的产品方向、目标用户、核心价值与边界。

---

## 2. 工作流程

1. 读取已确认的首批 `RQ-*`。
2. 提炼项目目标、用户群体、核心场景、非目标范围。
3. 若存在多个产品方向，先输出 3-5 个产品定位方案并推荐最优。
4. 人类确认后，写入 `Base/00-项目总览/`。
5. 更新相关 `RQ/需求状态.md`，标注已纳入产品基线。

---

## 3. 输出目录

```
Base/00-项目总览/
├── 项目目标.md
├── 产品状态看板.md
├── 阶段进度.md
└── 决策记录.md
```

详细产物模板见：`./PRODUCT-TEMPLATE.md`

---

## 4. 项目目标.md 模板

```
# 项目目标

## 1. 产品愿景

## 2. 目标用户

## 3. 核心问题

## 4. 核心价值

## 5. 首批需求对象
- RQ-xxx：

## 6. 非目标范围

## 7. 成功标准
```

---

```

## 当前阶段

## 已确认需求

## 待确认需求

## 当前设计中对象

## 当前实现中对象

## 当前风险

## 下一步建议
```

---

## 6. 禁止事项

- 禁止将所有需求写入一个 `产品概述.md`。
- 禁止无 `RQ` 直接生成产品基线。
- 禁止 AI 自行改变已确认产品方向。
- 禁止产品基线与 RQ 状态不一致。
