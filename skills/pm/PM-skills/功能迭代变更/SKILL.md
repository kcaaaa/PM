# 功能迭代变更 · CR 变更对象包

> 需求可以随时变化，但变化不能直接覆盖原始需求。所有变化必须进入 CR 对象包。

---

## 1. 输入

- 用户反馈、体验偏差、新想法、测试问题或 AI 实现偏差。
- 原始 `RQ / PD / TD / FT / IMP / TEST` 链路。

---

## 2. 工作流程

1. 定位变更影响的上级对象：`RQ / PD / TD / FT / IMP / TEST`。
2. 若无法定位来源，先补充反馈记录，不直接修改代码。
3. 创建 `CR-*` 变更包。
4. 输出 3-5 个处理策略：接受、延后、拆分、替代、拒绝等。
5. 综合人类意图、用户价值、复杂度、开发量、代码适配度、风险推荐最优策略。
6. 人类确认后，更新受影响对象状态，并创建增量 `IMP-*` 与回归 `TEST-*`。

---

## 3. 输出对象包

```
Base/08-变更管理/CR-xxx-变更名称/
├── 变更说明.md
├── 影响分析.md
├── AI处理建议.md
├── 人类决策记录.md
└── 变更执行记录.md
```

详细产物模板见：`./CR-TEMPLATE.md`

---

## 6. 禁止事项

- 禁止直接改原始需求文档来掩盖变化。
- 禁止无影响分析就进入实现。
- 禁止没有人类决策记录就执行变更。
- 禁止把变更当作普通 BUG 静默修复。
