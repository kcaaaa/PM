# 功能任务规划 · IMP 实现包

> 任务规划的目标不是生成一张大任务表，而是为每个功能创建独立实现包。

---

## 1. 输入

- `RQ-*` 需求包。
- `PD-*` 产品设计包。
- `TD-*` 技术方案包。
- `FT-*` 功能包及 `验收标准.md`。

---

## 2. 工作流程

1. 检查上级链路：`RQ → VR → PD → TD → FT` 是否完整。
2. 若缺少验收标准，不得创建实现任务。
3. 将功能拆分为数据层、接口层、前端层、测试层、集成层任务。
4. 每个任务必须标注对应 `FT` 与验收标准。
5. 创建 `IMP-*` 实现包。
6. 进入编码实现前，输出本次实现范围与不做范围。

---

## 3. 输出对象包

```
Base/06-实现管理/IMP-xxx-功能名称实现/
├── 实现任务清单.md
├── 数据层实现.md
├── 接口层实现.md
├── 前端实现.md
├── 测试实现.md
└── 实现完成记录.md
```

详细产物模板见：`./IMP-TEMPLATE.md`

---

## 5. 禁止事项

- 禁止无 `FT/验收标准.md` 创建实现任务。
- 禁止把多个不相关功能混入一个 IMP 包。
- 禁止任务无法追溯到验收标准。
- 禁止未说明不做范围就进入编码。
