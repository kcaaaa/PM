# 知识库 · 06 图表选型（25 种数据可视化方案）

> **来源**：由 `charts.csv` 完整转译。
> **作用**：根据数据类型选出**最佳图表 + 库推荐 + 色彩策略 + 无障碍建议**。
> **搜索字段**：`Data Type / Keywords / Best Chart Type / Accessibility Notes`

---

## 使用决策树（用户场景 → 图表）

```
需要显示……
├── 时间趋势 / 增长 / 时间线       → #1 Line Chart（或 Area Chart）
├── 分类对比 / 排名                 → #2 Bar Chart（Horizontal / Vertical）
├── 占比 / 份额 / 百分比            → #3 Pie / Donut（≤ 5-6 片）或 #19 Waffle Chart
├── 相关性 / 分布 / 模式            → #4 Scatter / Bubble Chart
├── 强度 / 密度 / 矩阵              → #5 Heat Map / Choropleth
├── 地理数据 / 区域                 → #6 Choropleth Map / Bubble Map
├── 漏斗 / 流程转化                 → #7 Funnel / Sankey
├── 绩效 vs 目标                    → #8 Gauge / Bullet Chart
├── 时间序列预测                    → #9 Line + Confidence Band
├── 异常检测                        → #10 Line Chart with Highlights
├── 层级 / 嵌套数据                 → #11 Treemap / Sunburst
├── 流量/流向数据                   → #12 Sankey / Chord Diagram
├── 累计变化 / 增减                 → #13 Waterfall Chart
├── 多维度对比                      → #14 Radar / Spider Chart
├── 股票 / OHLC                     → #15 Candlestick
├── 关系/连接图谱                   → #16 Network Graph
├── 统计分布（分位数 / 异常值）     → #17 Box Plot / Violin Plot
├── 紧凑型绩效 vs 目标              → #18 Bullet Chart
├── 比例 / 百分比（替代饼图）       → #19 Waffle Chart
├── 层级比例                        → #20 Sunburst Chart
├── 根因分析 / 决策分解             → #21 Decomposition Tree
├── 3D / 空间 / 分子/地形           → #22 3D Scatter / Surface Plot
├── 实时流                          → #23 Streaming Area Chart
├── 情感 / NLP                      → #24 Word Cloud with Sentiment
└── 流程挖掘 / 瓶颈                 → #25 Process Map / Graph
```

## Part B · 库选择指南

| 库 | 适合场景 | 不适合 |
|---|---|---|
| **Chart.js** | 常规 Line/Bar/Pie/Radar/Bubble；快速接入；移动友好 | 复杂交互 / 3D / 大数据量（>10k 点） |
| **Recharts** | React 项目；Line/Bar/Area/Pie/Radar/Funnel；声明式 | 大型数据集 / 高度定制 |
| **ApexCharts** | 实时更新；Candlestick；Heat Map；Streaming | 3D / 超大数据集 |
| **D3.js** | 高度定制；科研级；所有图表 | 快速上线；入门简单 |
| **Plotly** | 科学可视化；3D；Box Plot；分布 | 极端性能敏感场景 |
| **Mapbox / Leaflet** | 地理/地图 | 非地图场景 |
| **Lightweight Charts (TradingView)** | 金融 Candlestick / 高频交易 | 非金融场景 |
| **Three.js / Deck.gl** | 3D / WebGL | 2D 常规图表 |
| **d3-force / Vis.js / Cytoscape.js** | 关系图 / 网络图 | <500 节点不需要 |
| **React-Flow** | 流程 / 决策树 / 工作流 | 传统图表 |
| **D3-cloud / Highcharts / Nivo** | Word Cloud / 统计 / 企业报表 | 轻量场景 |

## Part C · 颜色策略总则

1. **类别 ≤ 6**：超过时用 `#OTHERS` 合并或改用其他图表。
2. **连续数据用单色渐变**：浅→深（蓝色系：`#BAE6FD` → `#1E3A8A`）。
3. **发散数据（正负）用双端渐变**：`#EF4444` → `#FAFAFA` → `#10B981`。
4. **强调值加 pattern overlay**（条纹 / 圆点）——服务色盲用户。
5. **暗色主题**：Grid 用 `#334155` 或更淡；Axis 用 `#E2E8F0` 或更白。
6. **品牌色优先**：主系列用 Primary；次要系列在同 Hue 做色阶变化。

## Part D · 无障碍红线

- 所有图表**必须**在 `aria-label` 中给出摘要。
- 时间序列要有 **`<table>` 数据表等价物**（visually hidden，供屏幕阅读器）。
- **色盲友好**：不仅用颜色区分系列；加 line-style / pattern / shape 辅助。
- **实时动画**：提供 Pause 按钮，尊重 `prefers-reduced-motion`。
- **键盘可达**：图表节点支持 `Tab` 聚焦 + `Enter/Space` 激活。
- **提供数据表下载 / CSV 导出**，符合 WCAG 2.1 AA 的 "替代方式"要求。
