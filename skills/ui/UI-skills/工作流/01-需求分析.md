# 工作流 Step 1 · UI 需求分析

> **职责**：把用户的 UI/UX 请求解析为可用于设计判断的信号，并绑定到 Base 对象包链路。

---

## 1. Base 关联检查

开始 UI 分析前，先判断当前请求对应的 Base 对象：

| 场景 | 必要关联 |
|---|---|
| 页面流程 / 信息架构 / 交互规则 | `RQ-*`、`PD-*` |
| 功能组件 / 状态 / 异常交互 | `RQ-*`、`FT-*` |
| 技术落地 / 组件代码 / 样式实现 | `RQ-*`、`TD-*`、`FT-*`、`IMP-*` |
| 反馈偏差 / UI 修改 | `FB-*` 或 `CR-*` |
| 纯风格咨询 | 可不绑定，但不得直接进入实现 |

若缺少必要对象包，应先返回 PM 链路补齐，不得直接输出实现代码。

---

## 2. UI 四元组提取模板

| 字段 | 说明 | 示例值 |
|---|---|---|
| `product_type` | 产品 / 行业大类，用于查 `知识库/01` | SaaS、E-commerce、Healthcare、Fintech、Dashboard |
| `style_keywords` | 用户想要的气质词，用于二次排序 | minimal、elegant、playful、professional、dark |
| `industry` | 细分行业，用于微调推理规则 | 医疗、教育、政府、金融、儿童 |
| `stack` | 技术栈；未指定时不直接默认实现，只作为建议 | html-tailwind、react、nextjs、vue、shadcn、flutter |

---

## 3. 分析规则

### 3.1 从显式词抽取 `product_type`

| 触发词 | product_type |
|---|---|
| SaaS / 企业软件 / B2B 云 | `SaaS (General)` |
| 电商 / 商城 / shop / store | `E-commerce` |
| 金融 / 银行 / 交易 / crypto | `Fintech/Crypto` 或 `Banking/Traditional Finance` |
| 医疗 / 诊所 / 健康 App | `Healthcare App` 或 `Medical Clinic` |
| 教育 / 网课 / bootcamp | `Online Course/E-learning` |
| 政府 / 公共服务 | `Government/Public` |
| 仪表盘 / dashboard / analytics | `Analytics Dashboard` |
| AI / Chatbot / LLM | `AI/Chatbot Platform` |

完整映射见《知识库/01-产品-风格映射.md》。

### 3.2 抽取 `style_keywords`

- 简约系：`minimal / clean / 简洁 / 极简 / spacious`
- 优雅系：`elegant / luxury / premium / 高端 / 精致`
- 活泼系：`playful / vibrant / 活泼 / 鲜艳`
- 专业系：`professional / corporate / 专业 / 商务`
- 暗色系：`dark / OLED / 深色 / 暗黑`
- 技术系：`futuristic / cyber / tech / 科技感`
- 有机系：`organic / nature / earthy / 自然`
- 怀旧系：`retro / vintage / nostalgic / 复古`

### 3.3 解析 `stack`

| 显式线索 | stack |
|---|---|
| HTML / 纯前端 / 静态页 | `html-tailwind` |
| React / JSX / hooks | `react` |
| Next.js / app router / SSR | `nextjs` |
| Vue / SFC / composition | `vue` |
| Nuxt UI | `nuxt-ui` |
| shadcn / Radix UI | `shadcn` |
| SwiftUI / iOS / macOS | `swiftui` |
| React Native / Expo | `react-native` |
| Flutter / Dart | `flutter` |

技术栈只作为设计落地参考；只有 `IMP-*` 已确认时，才允许进入代码产出。

---

## 4. 输出模板

```yaml
base:
  rq: RQ-xxx 或 无
  pd: PD-xxx 或 无
  td: TD-xxx 或 无
  ft: FT-xxx 或 无
  imp: IMP-xxx 或 无
  cr: CR-xxx 或 无
ui_signal:
  product_type: <...>
  style_keywords: [<...>]
  industry: <...>
  stack: <...>
next_target: PD / TD / FT / IMP / CR / 咨询建议
```

## 5. 完成后

- 产品设计类 → 转《工作流/02-设计系统生成.md》，产出写入 `PD-*`。
- 技术方案类 → 转《工作流/02》+《工作流/04》，产出写入 `TD-*`。
- 实现类 → 确认 `IMP-*` 后再转《工作流/04》。
