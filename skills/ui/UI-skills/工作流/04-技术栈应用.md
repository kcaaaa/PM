# 工作流 Step 4 · 技术栈应用

> **职责**：把已确认的设计系统映射到具体技术栈。只有关联 `IMP-*` 且人类明确要求实现时，才输出或修改代码。

---

## 1. 前置条件

| 输出类型 | 必须具备 |
|---|---|
| 技术建议 | `RQ-*`、`PD-*`、`TD-*` |
| 实现建议 | `RQ-*`、`PD-*`、`TD-*`、`FT-*`、`IMP-*` |
| 直接代码产出 | 已确认 `IMP-*` + 用户明确要求实现 |
| UI 变更实现 | `CR-*` + 增量 `IMP-*` |

缺少 `IMP-*` 时，只能给设计或技术建议，不得输出完整代码。

---

## 2. 栈选择决策

| 场景 | 推荐栈 |
|---|---|
| 纯静态页面 | `html-tailwind` |
| React / JSX / Hooks | `react` |
| Next.js / app router / SSR / SEO | `nextjs` |
| Vue 3 / SFC / Composition API | `vue` |
| Nuxt 3 | `nuxtjs` |
| Nuxt UI | `nuxt-ui` |
| Svelte / SvelteKit | `svelte` |
| Astro / Islands | `astro` |
| shadcn / Radix | `shadcn` |
| SwiftUI / iOS / macOS | `swiftui` |
| React Native / Expo | `react-native` |
| Flutter / Dart | `flutter` |
| Android / Jetpack Compose | `jetpack-compose` |

冲突时优先更具体的栈，如 `shadcn > react > html-tailwind`。

---

## 3. 映射规则

1. 颜色：映射到主题对象、CSS Variables、Tailwind Config 或设计 Token。
2. 字体：映射到全局样式、字体导入、Tailwind Config 或平台主题。
3. 圆角 / 阴影 / 间距：映射到统一设计变量。
4. 动效：映射到符合 150-300ms 的过渡与 reduced-motion 支持。
5. 组件：映射为对应技术栈的组件 API。
6. 性能：查对应技术栈文档的 Performance 段。

---

## 4. 输出到 Base

| 内容 | 写入位置 |
|---|---|
| 栈选择理由 | `TD-*/方案说明.md` |
| 主题变量 | `TD-*/方案说明.md` 或 `IMP-*/前端实现.md` |
| 组件落地策略 | `TD-*/接口设计.md` / `IMP-*/前端实现.md` |
| 代码任务拆分 | `IMP-*/实现任务清单.md` |
| 验证清单 | `TEST-*/测试用例.md` |

---

## 5. 代码产出限制

写代码前必须再次确认：

- 当前 `IMP-*` 存在。
- `IMP/实现任务清单.md` 已包含前端实现任务。
- `FT/验收标准.md` 已包含 UI 验收标准。
- 修改范围不会改变产品意图；如会改变，先创建 `CR-*`。
