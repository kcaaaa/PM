# 检查清单 · 交付前检查（可复制给用户）

> 任何 UI 代码 / 设计交付前，**逐条打勾**验证。
> 此清单对应原 UI-skill.md 的 "Pre-Delivery Checklist"，并做了扩展。

---

## 视觉质量（Visual Quality）

- [ ] 没有用 Emoji 当 icon（使用 SVG：Heroicons / Lucide）
- [ ] 所有图标来自同一 icon 集（不要 Heroicons 和 Lucide 混用）
- [ ] 品牌 logo 准确（从 Simple Icons 官方取）
- [ ] Hover 态不会导致 layout shift（不用 `scale`）
- [ ] 主题色直接用（`bg-primary`），不是 `var()` 套壳
- [ ] 同类组件圆角、阴影、内外边距保持一致
- [ ] 所有色值来自《03-配色方案》对应产品行，或其确定的回退

## 交互（Interaction）

- [ ] 所有可点击元素带 `cursor-pointer`
- [ ] Hover 态有明确视觉反馈（color / shadow / border）
- [ ] 过渡平滑（150-300ms），不瞬时切换，不超 500ms
- [ ] 所有按钮 / 链接 / 可聚焦元素**可见焦点环**（keyboard nav）
- [ ] 按压态（active）有反馈（`active:scale-95` 或同等）
- [ ] 禁用态明显（`opacity-50 cursor-not-allowed`）
- [ ] 重要 / 不可逆操作前弹 `confirm()` 或自定义确认对话

## Light / Dark Mode 对比

- [ ] 浅色模式正文对比度 ≥ 4.5:1
- [ ] 玻璃 / 半透明元素在浅色下**可见**（`bg-white/80+`，不是 `bg-white/10`）
- [ ] 边框在两种模式下**都可见**（`border-gray-200` / `dark:border-white/10`）
- [ ] 深 / 浅 两种主题都已测试
- [ ] 图标 / icon 在 dark mode 下对比度依然达标

## 布局（Layout）

- [ ] 浮动元素（navbar、浮窗）离边缘有合适间距（`top-4 left-4 right-4`）
- [ ] 没有内容被 fixed navbar 遮挡
- [ ] 响应式测试 375 / 768 / 1024 / 1440 四档
- [ ] 移动端没有横向滚动（`overflow-x-hidden` 或 `max-w-full`）
- [ ] 所有图片 / iframe 有固定 `aspect-ratio` 或 `width/height`（防 CLS）
- [ ] Touch targets 移动端 ≥ 44×44px
- [ ] 间距使用系统 scale（`p-4 m-6 gap-8`），避免 `p-[15px]` 随意像素

## 字体与排印（Typography）

- [ ] 字体从《04-字体搭配》对应组合导入（`@import` or Tailwind Config）
- [ ] `font-display: swap` 确保无 FOIT
- [ ] 正文 line-height 1.5-1.75
- [ ] 正文行长 65-75ch（`max-w-prose`）
- [ ] 移动端正文字号 ≥ 16px
- [ ] 标题与正文有明显层级差（粗度 / 大小）

## 表单（Forms）

- [ ] 每个 input 都有 `<label for="...">`
- [ ] 错误文本紧靠出错字段
- [ ] 提交按钮显示 loading 态（spinner + disabled）
- [ ] 密码字段有显/隐切换
- [ ] `type="email"` / `type="tel"` / `type="number"` 等正确
- [ ] `autocomplete` 属性正确（`email` / `current-password` / `one-time-code` 等）
- [ ] 不阻止粘贴（`onPaste preventDefault` 禁用）
- [ ] 必填字段有视觉标记（星号 / required 文字）

## 图片 / 媒体（Media）

- [ ] 所有图片有 `alt` 属性（装饰用 `alt=""`）
- [ ] 首屏之下 `loading="lazy"`
- [ ] 使用 WebP / AVIF 优化格式
- [ ] `<Image>`（Next/Nuxt）或 `<img srcset>` 多尺寸
- [ ] 视频无自动播放（除非静音并位于交互区）

## 无障碍（Accessibility）

- [ ] 所有 `<img>` 有 alt
- [ ] 图标按钮有 `aria-label`
- [ ] 装饰图标 `aria-hidden="true"`
- [ ] 表单输入有 label（见上）
- [ ] 动态更新区域 `aria-live="polite"`（或 `role="alert"`）
- [ ] 颜色不是唯一信息（红绿色盲友好：加图标 / 文字）
- [ ] 键盘 Tab 顺序合理
- [ ] 所有交互支持键盘（Enter / Space）
- [ ] `prefers-reduced-motion` 被尊重
- [ ] Skip-to-main link 存在（多导航站点）

## 性能（Performance）

- [ ] 没有瀑布流（`Promise.all` / `Suspense` 边界）
- [ ] 大组件用 dynamic import（`lazy()` / `next/dynamic`）
- [ ] 大列表虚拟化（> 100 项）
- [ ] 图片合理尺寸（不把 4000px 图展示为 400px）
- [ ] Preconnect 第三方 CDN
- [ ] 不用 barrel imports（直接深路径）
- [ ] Bundle size 监控（analyzer）

## SEO / Meta（Web 场景）

- [ ] `<title>` / `<meta name="description">` 有内容且不超 160 字
- [ ] OpenGraph 完整（`og:title` / `og:image` / `og:description`）
- [ ] Twitter Card meta
- [ ] 语义化标签（`<nav>/<main>/<article>/<footer>`）
- [ ] `<h1>` 唯一、与页面主题一致
- [ ] Sitemap / robots.txt 齐全

## 设计系统落地（Design System）

- [ ] 所有颜色、字体、间距、阴影都源自《03-配色方案》+《04-字体搭配》+《02-风格库》
- [ ] 《01-产品-风格映射》的 `Anti_Patterns` 全部规避
- [ ] 《01-产品-风格映射》的 `must_have` Decision Rules 全部满足
- [ ] 《07-UX 规范》的 High 级规则全部通过

## 代码质量（Code Quality）

- [ ] TypeScript 类型完整（Props / State / Events）
- [ ] 没有 `any`（除非明确注释原因）
- [ ] 组件单一职责 + 复用
- [ ] 没有被注释掉的死代码
- [ ] Linter / Formatter 通过（ESLint / Prettier）
- [ ] 没有 `console.log` 残留
