# UI-design-skill

`UI-design-skill` 是服务于 `Base` 对象包链路的 UI/UX 领域技能包，用于在产品设计、技术方案、功能验收、实现任务和测试验收中提供设计系统、交互规则、视觉规范与技术落地建议。

---

## 项目定位

- Base 约束：UI 建议必须绑定 `RQ / PD / TD / FT / IMP / CR / TEST`。
- 领域能力：覆盖 SaaS、电商、医疗、金融、作品集、Dashboard 等常见场景。
- 候选方案：重要 UI 决策应提供 2-4 个方案并推荐当前最优。
- 实现边界：只有 `IMP-*` 已确认且用户明确要求实现时，才输出或修改代码。
- 低门槛接入：不依赖脚本，直接读取 Markdown 即可运行。


## 标准使用流程

1. 阅读 `UI-skill.md` 判断当前 UI 请求属于咨询、产品设计、技术方案、实现建议还是变更影响。
2. 在 `UI-skills/工作流/01-需求分析.md` 提取 UI 四元组与 Base 关联对象。
3. 按 `UI-skills/工作流/02-设计系统生成.md` 生成 2-4 个候选设计方案并推荐最优。
4. 如需补充规则，使用 `UI-skills/工作流/03-补充搜索.md` 定向查询。
5. 在 `UI-skills/工作流/04-技术栈应用.md` 生成技术落地建议；只有 `IMP-*` 已确认时才进入代码产出。
6. 使用 `UI-skills/检查清单/通用UI规范.md` 与 `交付前检查.md` 对齐 `FT` 验收标准和 `TEST` 测试项。

## 输出契约

项目建议按以下顺序输出：

1. Base 关联：`RQ / PD / TD / FT / IMP / CR / TEST`。
2. UI 需求解读：产品类型、行业、风格关键词、技术栈。
3. 候选设计方案：2-4 个方向与取舍。
4. 推荐设计系统：Pattern / Style / Colors / Typography / Effects / Anti-patterns。
5. 应写入的 Base 对象路径。
6. 代码限制：是否允许进入 `IMP-*` 实现。

---

## 默认约束

- 不用 Emoji 当 UI 图标，使用 SVG 图标库。
- 所有可点击元素要有 `cursor-pointer`。
- Hover 不应引发布局抖动。
- 浅色模式正文对比度至少 4.5:1。
- 过渡建议 150-300ms。
- 必须支持 `prefers-reduced-motion`。
- 必测响应式断点：375 / 768 / 1024 / 1440。
- UI 方案无法追溯到 Base 时，只能作为咨询建议，不能进入实现。

---

## 适用场景

- 生成或评审 `PD-*` 中的页面流程和交互规则。
- 为 `TD-*` 提供 UI 技术落地方案。
- 为 `FT-*` 补充 UI 验收标准。
- 为 `IMP-*` 提供前端实现建议。
- 为 `TEST-*` 补充 UI 测试用例。
- 为 `CR-*` 做 UI 变更影响分析。
