# AI Agents CLI 技能使用场景介绍

## 该技能应该在何时使用

本技能是 AI 原生产品协作的总入口，围绕 Base 项目认知数据库，覆盖从需求提出到测试验收的完整链路。以下场景触发本技能的调用：

---

## 一、项目启动与初始化

当用户想要从零开始一个新产品或新项目时，本技能负责建立 Base 项目认知数据库，初始化项目结构，并按链路推进。

**典型触发语句：**

- "我想做一个 XXX 产品，怎么开始？"
- "帮我初始化一个新项目"
- "这个项目该怎么组织？"

**执行内容：**

- 生成项目目标、产品状态看板、阶段进度
- 建立 Base 目录结构（00-项目总览至 09-测试验收）
- 按 RQ → VR → PD → TD → FT → IMP → TEST 链路推进

---

## 二、需求管理与澄清

当用户提出新需求、新功能想法，或需要梳理、变更已有需求时，本技能负责捕获需求、生成候选方案、经人类确认后写入 Base。

**典型触发语句：**

- "我想加一个 XXX 功能"
- "这个需求该怎么实现？"
- "帮我梳理一下当前的需求"
- "之前做的功能需要改一下"

**执行内容：**

- 识别意图与当前 Base 状态
- 提出 3-5 个候选方案并给出推荐
- 经人类选择、修正或否定后，写入 RQ-* 对象包
- 按链路补齐 VR、PD、TD、FT 等缺失环节

---

## 三、产品设计与方案评估

当用户需要 AI 提供多个候选方案并评估取舍，而非直接进入实现时，本技能负责生成方案、评估维度、推荐最优解。

**典型触发语句：**

- "帮我设计一下这个功能"
- "这几个方案哪个更好？"
- "该怎么设计这个页面流程？"
- "技术方案怎么选？"

**执行内容：**

- 评估人类意图匹配度、用户价值、实现复杂度、代码开发量、代码适配度、风险与扩展性
- 输出推荐方案及理由
- 经人类确认后写入 PD-* 或 TD-* 对象包

---

## 四、功能实现与编码

当需求已确认，需要按链路推进到技术设计、编码实现、测试验收时，本技能负责创建实现任务、执行代码修改、记录验证结果。

**典型触发语句：**

- "开始实现这个功能"
- "帮我写一下这个接口"
- "前端页面怎么实现？"
- "数据库表怎么设计？"

**执行内容：**

- 创建 IMP-* 实现对象包，记录任务清单、数据层、接口层、前端、测试实现
- 按子技能路由：编码走 Code Skill，数据库走 DB Skill，UI 走 UI Skill
- 修改代码前声明当前对应的 RQ / FT / IMP
- 所有实现任务追溯到需求、设计、方案、验收标准

---

## 五、项目结构咨询

当用户询问项目结构、文件关系、Base 目录组织方式时，本技能负责解释 Base 与代码工程目录的关系。

**典型触发语句：**

- "项目里的 Base 目录是做什么的？"
- "Base 和代码目录是什么关系？"
- "IMP-* 里该写什么？"
- "需求该放在哪个目录？"

**执行内容：**

- 解释 Base 是 AI 的项目认知数据库，不存放源码
- 说明 Base/06-实现管理/IMP-* 是 Base 与代码之间的桥梁
- 指导对象包命名规则与目录结构

---

## 六、反馈与变更处理

当实现结果与预期有偏差，或需要修改已有功能时，本技能负责创建变更对象包、分析影响、经人类决策后执行。

**典型触发语句：**

- "这里和我想的不一样"
- "我想改一下之前做的功能"
- "这个逻辑需要调整"
- "能不能换成另一种方式？"

**执行内容：**

- 创建 FB-* 反馈对象包记录偏差
- 创建 CR-* 变更对象包，包含变更说明、影响分析、AI 处理建议、人类决策记录、变更执行记录
- 修改已有功能必须创建 CR-*，不得直接覆盖原需求意图

---

## 七、Bug 修复与测试问题

当已有功能出现异常，需要定位问题并修复时，本技能负责走 TEST / BUG 修复链路。

**典型触发语句：**

- "这里有个 Bug，帮我修一下"
- "测试报错了"
- "这个功能不正常"
- "帮我写一下测试用例"

**执行内容：**

- 定位问题，创建或更新 TEST-* 测试对象包
- 记录测试用例、测试记录、问题清单、验收结论
- 按 Bug 修复工作流执行，记录修复过程与验证结果

---

## 八、多 Agent 协作调度

当任务涉及产品、代码、数据库、UI 多个领域，需要统一调度时，本技能作为总入口，按需挂载子 Skill。

**典型触发语句：**

- "这个功能涉及前端和后端，一起做吧"
- "帮我设计数据库和页面"
- "从产品到实现完整做一遍"

**子 Skill 挂载规则：**

| 场景 | 挂载子 Skill |
|---|---|
| 产品 / 需求 / 变更 | skills/pm/PM-skill.md |
| 编码 / 实现 / 修复 | skills/code/code-skill.md |
| 数据库 / 数据结构 / SQL | skills/db/DB-skill.md |
| UI / 页面 / 交互 / 视觉 | skills/ui/UI-skill.md |

---

## 九、跨平台配置安装

当用户需要在 Cursor、Trae、Continue、Claude Code 等 IDE 中安装 Agent 配置时，本技能提供 CLI 工具完成安装与维护。

**典型触发语句：**

- "怎么在 Cursor 里安装这个 Agent？"
- "帮我配置一下 Trae"
- "继续 IDE 的 Agent 怎么装？"
- "更新一下 Agent 配置"

**执行内容：**

- 使用 ai-agents-cli install 命令安装指定平台配置
- 支持 --global 全局安装或项目级安装
- 提供 doctor、check、status、update、uninstall 等维护命令

---

## 核心判断标准

只要用户的意图涉及以下任一维度，即应调用本技能：

1. **产品意图层面** — 新需求、功能变更、设计方向
2. **实现执行层面** — 编码、测试、Bug 修复
3. **项目管理层面** — 方案评估、进度跟踪、链路推进
4. **工具配置层面** — 多平台 Agent / Skill 安装与维护
